JavaScript is single-threaded (it can only do one thing at a time). But we still need to handle things like:

  • User interactions (clicks, inputs)
  • Timers (setTimeout, setInterval)

ðŸ‘‰ The event loop is what lets JavaScript handle asynchronous tasks without blocking the main thread.

ðŸ”¹ Summary

  • JS runs synchronously by default.
  • Event loop allows async tasks by managing the call stack and task queues.
  • Microtasks (Promises) run before macrotasks (setTimeout) in each tick.

ðŸ”¹ How It Works

  • Call Stack: Where JavaScript executes code line by line. Functions get pushed when called, popped when finished.
  • Web APIs: When async functions like setTimeout or fetch are called, they are handed off to the browser (or Node.js APIs).
  • Callback / Task Queue: When an async task completes, its callback is placed into a queue (waiting to be executed).
  • Microtask Queue: Special queue for Promises (then/catch/finally, async/await). Microtasks run before normal tasks (callbacks).
  • Event Loop: Keeps checking:
    • If the call stack is empty â†’ it pushes tasks from the queue into the stack.
    • Always prioritizes microtasks before moving on to the next task.
